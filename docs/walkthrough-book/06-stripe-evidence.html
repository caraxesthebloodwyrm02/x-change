Stripe & EvidenceLedger

StripePaymentConfirmation

  • Primary event: payment_intent.succeeded.
  • Idempotency: stripe_event_id unique; duplicates acknowledged without double-apply.
  • Link keys: metadata.reward_id, metadata.student_id.
  • Missing metadata → support_signals.kind = stripe_missing_metadata, HTTP 200 to Stripe, no reward state mutation.
  • Mismatch → support signal; do not apply payment to ledger state.
