PSC

PrincipledServiceContract

Before a reward is interpreted, a service promise exists that students and support can read.

Field Meaning
What may be rewarded Observable progress (declared in contract JSON for v0).
Required evidence Minimum types — e.g. Glass session plus optional failure snapshot.
Post-success expectation Narrative in promise_json.
Stripe linkage payment_intent.succeeded with reward + student metadata confirms payment, not satisfaction.
Support path Mismatches emit support_signals; no silent drops.
Fulfillment confirmation Student acknowledgement is separate from Stripe.

Default contract id: psc-v0-default (seeded on first init).
