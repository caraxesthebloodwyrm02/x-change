scope-integration.md

Token + tool + evidence

Integration is emergent: shared reward_id on the ledger, no pre-declared tool↔token map.

Tool Sources (Glass, GRID, Calculator)
        |
  Ingest / Adapter Layer
        |
  Evidence Ledger (evidence_ledger)
        |
  Reward Ledger (reward_ledger)
        |
  Token Scope (InsightTier, RarityScore, Exchange)

Linkage rules

  • Token → reward: issued at approval; attributes include tier, rarity, amount, issued_at.
  • Reward → evidence: optional reward_id; PATCH can link later.
  • Tool → reward: no auto-inference; explicit booleans drive transitions.
  • Tool → token: indirect via rewards only.

Validation anchors

EvidenceType enum, InsightTier, rarity clamp, ingest_bool(), exchange three-layer evaluation, XCHANGE_GRID_SUB_MAX_BYTES for substantiation payload size.
