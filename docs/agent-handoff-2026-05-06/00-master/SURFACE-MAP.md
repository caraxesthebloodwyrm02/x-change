## Current System Shape

The local ecosystem is a multi-root workspace:

- `x-change` is the current documentation and policy host for this handoff.
- `CascadeProjects` is the large multi-domain MCP and application surface.
- `GRUFF` is the trust/router/workspace family.
- `glass` is a co-presence and bridge surface.
- `personal-rag` and `.claude/knowledge` are the beginnings of memory infrastructure.
- `.claude`, `.codex`, `.cursor`, `.windsurf`, `.config`, `.gemini`, `.opencode`, and `.copilot` contain active coding-tool wiring.

## Known P0 Friction

- `npm` and `npx` exist inside nvm but are not on PATH for agents.
- Several MCP configs disagree on server coverage and command paths.
- Some configs still carry stale `/home/caraxes` assumptions.
- Two `hogsmade` systemd user timers point to a missing driver path.
- Snap-backed Go, Rust, Cargo, and Code commands are degraded.
- No `.rag_db` was found at the expected memory locations during the audit.

## Authority Direction

Do not solve the ecosystem by manually editing every client. Build authority files first, then generate or validate derived surfaces from them.
