# Directory, Domain, And Resurfacing Authority

## Objective

Prepare structural discipline, domain distribution, and buried asset resurfacing without breaking paths.

## Directory Authority

| Root | Classification | Rule |
|---|---|---|
| `/home/irfankabir/x-change` | handoff and policy host | keep handoff here until promotion is explicit |
| `/home/irfankabir/CascadeProjects` | active local root | verify before moving or consolidating |
| `/mnt/arch_data/home/caraxes/CascadeProjects` | preserved large workspace | treat as active/preserved based on live path use |

| `/home/irfankabir/canopy`, `/home/irfankabir/roots` | active or mirrored roots | classify before moving |
| archive directories | archive | never delete during authority buildout |

## Domain Authority

Each domain record should include:

- domain name
- real-world use case
- primary assets
- secondary assets
- missing pieces
- integration surfaces
- diagram candidates

Preserve multi-domain projects. Do not force a single label when a project spans education, finance, data, security, automation, or personal development.

## Resurfacing Authority

Each buried asset candidate should include:

- candidate name
- current path
- tier
- value statement
- current status
- target surface
- owner
- validation

## No-Break Rule

No directory move or resurfacing promotion is allowed until:

- source path exists
- target path is declared
- references are searched
- rollback note exists
- archive status is clear
